### Task 1: Fix formatValue for Small Decimals (Fatalities Per Capita)

**Files:**
- Modify: `web/src/lib/data.ts:736-752` (`formatValue` function)
- Modify: `web/src/lib/__tests__/data.test.ts`

**Problem:** `fatalities_per_capita` for Dallas County = ~0.0009. `formatValue(0.0009, "decimal")` returns `"0.0"` because it uses `toFixed(1)`. Per-capita rates below 0.05 display as "0.0", which looks like a data error.

**Solution:** In the `"decimal"` case, detect values < 0.1 and use enough precision to show at least one significant digit. Values ≥ 0.1 keep the current `toFixed(1)`.

```typescript
  expect(formatValue(0.045, "decimal")).toBe("0.045");
  expect(formatValue(0.005, "decimal")).toBe("0.005");
});

it("keeps toFixed(1) for normal decimals", () => {
  expect(formatValue(11.9, "decimal")).toBe("11.9");
  expect(formatValue(2.3, "decimal")).toBe("2.3");
  expect(formatValue(0.7, "decimal")).toBe("0.7");
  expect(formatValue(0.1, "decimal")).toBe("0.1");
```

- [ ] **Step 3: Fix formatValue decimal case**

In `web/src/lib/data.ts`, replace the `"decimal"` case in `formatValue`:

```typescript
case "decimal": {
  if (value === 0) return "0";
  const abs = Math.abs(value);
  if (abs >= 0.1) return value.toFixed(1);

  return value.toFixed(digits);
}
```

```typescript
/** Format rank with denominator: "3rd of 45" */
export function formatRank(
  county: CountyData,
  metric: MetricKey,
  allCounties: CountyData[],
): string {
  const valid = allCounties.filter((c) => {
    const v = c[metric];
    return v !== null && v !== undefined && !Number.isNaN(v as number);
  });
  const rank = computeRank(county, metric, allCounties);
  return `${ordinal(rank)} of ${valid.length}`;
}
```

### Task 8: Shareable URLs via Search Params

**Files:**
- Modify: `web/src/app/page.tsx`

**Problem:** URL never changes when users select a metro, metric, or granularity. Can't share or bookmark specific views.

**Solution:** Sync `selectedMetro`, `selectedMetric.key`, `granularity`, and `selectedYear` to URL search params using `window.history.replaceState` (no full page navigation). On load, read params to restore state.

- [ ] **Step 1: Add URL state reading on mount**

In `web/src/app/page.tsx`, add a useEffect that reads URL params on mount. Place after state declarations:

```typescript
// Read URL params on mount
useEffect(() => {
  const params = new URLSearchParams(window.location.search);

  const metroParam = params.get("metro");
  if (metroParam && METROS[metroParam]) {
    flyToMetro(metroParam);
  }

  const metricParam = params.get("metric");
  if (metricParam) {
    const found = METRICS.find((m) => m.key === metricParam);
    if (found) setSelectedMetric(found);
  }

  const granParam = params.get("scale");
  if (granParam === "metro" || granParam === "county" || granParam === "block_group") {
    setGranularity(granParam);
  }

  const yearParam = params.get("year");
  if (yearParam) {
    setSelectedYear(Number(yearParam) || null);
  }
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, []);
```

Import `METRICS` at the top if not already imported (it's already available from the data import).

- [ ] **Step 2: Add URL state writing on changes**

Add a useEffect that writes to URL when state changes. Place after the read effect:

```typescript
// Write state to URL
useEffect(() => {
  const params = new URLSearchParams();
  if (selectedMetro) params.set("metro", selectedMetro);
  if (selectedMetric.key !== METRICS[0].key) params.set("metric", selectedMetric.key);
  if (granularity !== "county") params.set("scale", granularity);
  if (selectedYear) params.set("year", String(selectedYear));

  const qs = params.toString();
  const url = qs ? `${window.location.pathname}?${qs}` : window.location.pathname;
  window.history.replaceState(null, "", url);
}, [selectedMetro, selectedMetric.key, granularity, selectedYear]);
```
